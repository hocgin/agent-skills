## 核心特性

- **资讯聚合**：将多条独立科技新闻整合为有机整体，而非简单罗列
- **结构紧凑**：标题、导语、正文、结尾形成完整叙事闭环
- **风格统一**：新闻报道体，客观冷静，信息密度高
- **一键输出**：输入新闻素材，直接生成 markdown 格式成品文章

## 文章结构规范

### 标题（H1）

用一句话概括全部新闻主题，长度控制在 40-80 字。要点：

- 多个主题用逗号分隔，形成排比节奏
- 使用冒号前后分层：前半部分列举具体事件，后半部分拔高主题
- 体现新闻事件的多元性和关联性
- 避免使用感叹号

**结构模板**：
```
从[事件A]到[事件B]，从[事件C]到[事件D]：[主题升华]
```

**示例**：
```
从"看见三维"的AI革命到重现宇宙诞生瞬间，从巨头模型竞速到史诗级数据安全危机：技术突破、商业博弈与风险暗涌交织
```

### 导语（一段话，紧跟标题）

用一段话（100-200字）概述全文内容。要点：

- 用分号连接各条新闻的核心信息，形成并列结构
- 提炼出各条新闻之间的内在联系
- 最后用一句话点明整体趋势或意义
- 导语中不重复标题原文，但要呼应标题主题

**结构模板**：
```
从[事件A简要]，到[事件B简要]；从[事件C简要]，到[事件D简要]——[总结性概括]。[主题升华句]。
```

### 正文条目

每条新闻为一个独立单元，从上到下依次为：

```
![图片alt文字](图片URL)

### 新闻标题（H3）

正文段落（1-3段，每段50-200字）
```

**图片要求**：
- 优先使用新闻原始配图
- 如无配图，可使用代表性图片
- alt 文字即新闻标题

**条目标题要求**：
- 使用 H3 级别
- 完整陈述新闻事件，包含关键实体名称
- 长度控制在 20-50 字
- 直接陈述事实，不加主观评价

**正文段落要求**：
- 新闻报道体，客观冷静
- 5W1H 原则：何人、何事、何时、何地、何因、如何
- 信息密度高，避免空泛描述
- 技术术语自然融入，不做过多解释
- 如有具体数据（日期、金额、数量），必须包含
- 引用来源需提及（如期刊名、机构名）

### 分隔符

每条新闻之间用 `---` 分隔，无需在最后一条后添加。

### 结尾

一句话互动引导：

```
如果你觉得这篇文章对你有帮助，欢迎点赞和分享！
```

## 写作流程

### 阶段1：素材收集

1. 获取用户提供的新闻素材列表
2. 每条素材应包含：标题、核心内容、来源、配图（可选）
3. 如素材不足，主动搜索补充

### 阶段2：主题提炼

1. 分析各条新闻的内在关联
2. 确定一个贯穿全文的主题线索
3. 主题线索可以是：技术趋势、行业竞争、社会影响等

### 阶段3：标题与导语

1. 根据主题线索构思标题
2. 标题需涵盖所有核心新闻事件
3. 撰写导语段落，建立叙事框架

### 阶段4：正文写作

1. 按逻辑顺序排列各条新闻（可按重要性、时间、主题分类等）
2. 逐条扩写为完整段落
3. 保持各条目之间风格一致

### 阶段5：审校输出

1. 检查标题是否涵盖所有条目
2. 检查导语是否准确概括全文
3. 检查正文信息是否准确完整
4. 检查格式是否符合模板规范
5. 输出 markdown 文件

## 写作风格指南

### 语调

- **客观冷静**：不做主观判断，不使用感叹号（结尾互动除外）
- **专业克制**：避免营销话术和过度渲染
- **信息驱动**：让事实说话，不堆砌形容词

### 用词

- 使用规范的中文书面语
- 技术术语保持原文（如 "JoyAI-Image-Edit"、"夸克-胶子等离子体"）
- 机构名称使用官方全称（首次出现时可加括号注明英文缩写）
- 数字使用阿拉伯数字，单位使用中文（如 "10 petabytes"、"6,000 个"）

### 段落

- 每段聚焦一个核心信息
- 段首可以是时间、机构名、事件概要
- 段末可补充背景信息或影响分析
- 段落之间不使用过渡句，保持独立

### 禁止事项

- 不使用 "据悉"、"据了解" 等模糊来源表述
- 不使用感叹号（结尾互动除外）
- 不在条目标题中使用标点符号结尾
- 不使用 emoji
- 不添加作者署名和日期（除非用户要求）

## 输出格式

输出为标准 markdown 文件：

- 文件名：`YYYY-MM-DD.主题关键词.md`
- 编码：UTF-8
- 换行：段落之间空一行
- 图片：使用远程 URL，不使用本地路径
