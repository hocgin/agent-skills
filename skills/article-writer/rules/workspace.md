# 工作区管理规则

> **目标**：为每篇文章创建独立的工作区，避免不同文章的文件互相干扰

## 核心原则

- **一文章一工作区**：每篇文章都有独立的工作区目录
- **命名规范**：使用"编号.文章标题"格式
- **自包含**：工作区包含该文章的所有文件和配置
- **可追溯**：通过编号可以快速找到历史文章

---

## 工作区命名规则

### 格式

```
编号.文章标题
```

### 示例

```
001.claude-code完全指南
002.2024年AI编程工具对比
003.rust内存安全深度解析
004.微服务架构最佳实践
```

### 编号规则

- **格式**：3位数字，从 001 开始
- **递增**：每次创建新文章时，编号自动 +1
- **连续**：不跳号，保持连续性
- **查找**：通过查找最大的编号来确定下一个编号

### 标题规则

- **简洁**：使用文章的核心标题，去除副标题
- **小写**：全部使用小写字母
- **连字符**：空格和特殊字符替换为连字符 `-`
- **可读**：保持中文标题的可读性

## 创建新工作区

### AI 操作步骤

#### 步骤 1: 确定编号

**操作**：
1. 列出所有现有工作区
2. 找到最大编号
3. 新编号 = 最大编号 + 1

**示例**：

```markdown
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
📂 现有工作区
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━

001.claude-code完全指南
002.2024年AI编程工具对比
003.rust内存安全深度解析

━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
✅ 下一个编号: 004
━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━━
```

#### 步骤 2: 确定工作区名称

**根据文章标题生成**：

- 输入："Claude Code 完全指南：从入门到精通"
- 处理：
  1. 提取主标题："Claude Code 完全指南"
  2. 转换为小写："claude-code完全指南"
  3. 组合编号："001.claude-code完全指南"

#### 步骤 3: 创建工作区目录

**AI 操作**：
```bash
# 创建工作区目录
mkdir -p "001.claude-code完全指南"

# 创建子目录
cd "001.claude-code完全指南"
mkdir -p images
mkdir -p _knowledge_base/raw
mkdir -p _knowledge_base/indexed
```

#### 步骤 4: 创建初始文件

**创建以下文件**：

```markdown
# article.config.md
# 根据文章需求生成配置文件

---

# brief.md
# 根据用户需求生成需求文档

---

# README.md（可选）
# 工作区说明文件
```

## 使用方式

### 命令

```markdown
# 创建新工作区
/workspace create "文章标题"

# 切换到指定工作区
/workspace switch 001

# 列出所有工作区
/workspace list

# 删除工作区（谨慎操作）
/workspace delete 001
```

### 自动工作区管理

**AI 在开始写作前自动**：

1. **检查工作区**：是否已有工作区？
   - **是**：切换到该工作区
   - **否**：创建新工作区

2. **确认工作区**：向用户确认工作区名称
   ```
   将创建工作区: 001.claude-code完全指南
   是否继续？[Y/n]
   ```

3. **初始化文件**：创建必要的初始文件

## 工作区生命周期

### 阶段 1: 创建

- 创建工作区目录
- 初始化配置文件
- 设置基本参数

### 阶段 2: 写作

- 在工作区内进行调研、写作、审校
- 所有文件保存在工作区内
- 不影响其他工作区

### 阶段 3: 完成

- 文章完成后，工作区归档
- 可选：将 final.md 复制到上级目录
- 工作区保留作为历史记录

### 阶段 4: 归档

- 已完成的工作区可以归档
- 归档位置：`_archived/` 目录
- 保留结构，便于后续查找

## 最佳实践

### 1. 工作区命名

- ✅ 好: `001.claude-code完全指南`
- ❌ 差: `claude-code`（缺少编号）
- ❌ 差: `1.claude-code`（编号不规范）

### 2. 文件组织

- 每个工作区自包含，不引用其他工作区的文件
- 图片放在工作区的 `images/` 目录
- 调研数据放在工作区的 `_knowledge_base/` 目录

### 3. 工作区清理

- 完成后的工作区可以保留（作为历史记录）
- 或者归档到 `_archived/` 目录
- 不要删除已发布文章的工作区

### 4. 工作区备份

- 定期备份工作区
- 使用 Git 管理工作区
- 重要节点打标签（tag）

## 常见问题

### Q1: 工作区应该放在哪里?

**A**:
- **根目录即为工作区目录**：工作区直接放在项目根目录下
- **结构**:
  ```
  article-writer/           # 根目录
  ├── 001.claude-code完全指南/
  ├── 002.2024年AI编程工具对比/
  └── 003.rust内存安全深度解析/
  ```

### Q2: 如何重命名工作区?

**A**:
```bash
# 重命名工作区目录
mv "001.旧标题" "001.新标题"

# 更新工作区内的配置文件
vim "001.新标题/article.config.md"
```

### Q3: 工作区编号可以跳号吗?

**A**:
- **不建议**: 保持编号连续性
- **特殊情况**: 如果删除了某个工作区，可以补号或跳号
- **推荐**: 补号，保持连续性

### Q4: 可以同时编辑多个工作区吗?

**A**:
- **可以**: 但需要明确当前工作区
- **建议**: 一次专注于一个工作区
- **工具**: 使用 `/workspace switch` 切换工作区
