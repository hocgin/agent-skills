# 图片嵌入规则

> **目标**：将生成的图片嵌入到文章的合适位置，确保图片与文字内容协调

## 核心原则

- **位置精准**：图片应放在相关段落的紧邻位置
- **格式统一**：使用统一的图片语法和样式
- **说明完整**：每张图片都应有清晰的说明文字
- **体验优化**：考虑不同平台的显示效果

### 3️⃣ 分析嵌入位置

**根据配图方案确定嵌入位置**：

| 图片 | 推荐位置 | 具体段落 | 嵌入时机 |
|------|---------|---------|---------|
| 封面图 | 文章开头 | 标题后、正文前 | 第一张 |
| 数据对比图 | 对应章节 | 该章节的段落之后 | 紧邻相关内容 |
| 流程图 | 流程说明 | 描述流程的段落之后 | 紧邻相关内容 |

| 金句图 | 结尾 | 结论部分 | 最后 |

### 4️⃣ 选择图片语法

- GitHub README
- 技术博客（Hugo、Hexo、Jekyll）

## 图片嵌入最佳实践

### 1. 图片密度

- **推荐**: 1图/500-800字
- **最少**: 1图/1000字
- **最多**: 1图/300字（避免图片过多）

### 2. 图片尺寸

- **横版图片**: 宽度 100%，高度自适应
- **方形图片**: 宽度 70-80%，居中显示
- **统一比例**: 尽量使用相同的宽高比（如 16:9）

### 3. 图片大小

- **单张图片**: < 300 KB
- **所有图片**: 总计 < 2 MB
- **移动优化**: 考虑使用响应式图片

### 4. 图片说明

- **必需**: 每张图都应有说明
- **位置**: 图片下方，居中对齐
- **样式**: 斜体，小字号，灰色
- **内容**: 描述图片内容，或补充图片信息

### 5. 分隔符

- **图片前后**: 使用空行分隔
- **可选**: 使用 `<br/>` 或 `---` 增加间距
- **避免**: 图片与文字紧贴（影响阅读）

## 常见问题

### Q1: 图片路径应该用相对路径还是绝对路径?

**A**:
- **本地编辑**: 使用相对路径 `./images/01-cover.png`
- **在线发布**: 使用绝对路径 `https://your-cdn.com/images/01-cover.png`
- **建议**: 本地编辑用相对路径，发布时转换为 CDN 链接

### Q2: 图片太大，如何优化?

**A**:
1. **压缩图片**: 使用 TinyPNG 压缩
2. **调整尺寸**: 限制最大宽度为 1200px
3. **使用 WebP 格式**: 比 JPEG 小 30%
4. **使用 CDN**: 自动压缩和格式转换

### Q3: 如何统一图片尺寸?

**A**:
1. **生成时统一**: 使用 wenyan-cli 时指定相同的宽高比
2. **后期调整**: 使用图片编辑工具统一裁剪
3. **CSS 控制**: 使用样式控制显示尺寸

### Q4: 图片不显示怎么办?

**A**:
1. **检查路径**: 确认图片路径正确
2. **检查文件**: 确认图片文件存在
3. **检查格式**: 确认图片格式支持（jpg/png/webp）
4. **检查大小**: 确认图片没有超过平台限制

### Q5: 如何让图片居中?

**A**:
```markdown
<!-- Markdown -->
<p align="center">
  <img src="./images/01-cover.png" alt="图片说明">
</p>

<!-- HTML -->
<div style="text-align: center;">
  <img src="./images/01-cover.png" alt="图片说明">
</div>
```
