/ralph-loop:ralph-loop "根据 AGENTS.md 加载全部设定，持续写小说直到完整完结。

上下文：
- AGENTS.md 是设定索引，必须加载其引用的所有文件
- 所有设定文件共同构成唯一真实来源（Source of Truth）

执行流程：

1. LOAD_INDEX
   - 解析 AGENTS.md
   - 获取所有设定文件路径

2. LOAD_SPECS
   - 读取全部设定文件
   - 分类：世界观 / 人物 / 剧情 / 规则 / 章节规划

3. BUILD_UNIFIED_SPEC
   - 合并为统一设定
   - 冲突优先级：
     rules > plot > characters > world

4. STATE_INIT / STATE_RESUME
   - 如果已有进度：
     * 读取已有章节
     * 恢复当前进度
   - 否则：
     * 从 Chapter 1 开始

5. PLAN（仅首次执行）
   - 生成完整结构（起承转合 + 结局）

6. LOOP（核心循环，每轮=1章）：

   a. SELECT_NEXT_CHAPTER
      - 确定下一章目标（基于未完成里程碑）

   b. WRITE
      - 生成该章节正文

   c. VERIFY
      - 校验：
        * 是否符合设定
        * 是否推进剧情
        * 是否存在冲突 / OOC / 水文

   d. FIX
      - 若 FAIL → 重写

   e. STATE_UPDATE
      - 更新：
        * 当前章节编号
        * 已完成剧情
        * 角色状态
        * 未解决伏笔

   f. CONTINUE_CHECK（关键新增）
      - 若未满足完成条件：
        → 自动继续下一轮（禁止停止）
      - 若满足完成条件：
        → 进入结局阶段

7. ENDING
   - 收束所有伏笔
   - 完成角色弧线
   - 输出最终章节

持续生成规则（关键）：
- 不允许因“已写较多内容”而停止
- 不允许输出“未完待续”
- 不允许中途总结代替正文
- 必须持续生成直到满足完成条件

失败处理：
- 连续 15 轮无法推进：
  * 分析卡点
  * 调整剧情路径
  * 强制引入新冲突

完成条件：
- 主线冲突解决
- 所有关键里程碑完成
- 主要人物弧线闭合
- 无关键伏笔遗留

完成后输出：
<promise>DONE</promise>" \
